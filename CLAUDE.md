## Project Overview

This repository is Hugo's blog at https://rascunhos.blog — an **Astro** static site with a **Keystatic** CMS, deployed on **Vercel**.

- Posts are markdown files in `src/content/posts/*.md` (the source of truth).
- Public pages prerender to static; the Keystatic admin (`/keystatic`) + its API run on-demand.
- The design is plain CSS reused verbatim in `public/styles/styles.css`.

## Commands

```bash
npm install
npm run dev      # local dev server (http://localhost:4321); editing at /keystatic uses LOCAL storage (no auth)
npm run build    # production build (Astro + Vercel adapter); runs content-collection (Zod) validation
npm run preview  # preview the production build locally

# Run the GitHub-mode Keystatic setup wizard locally (only needed once, already done):
PUBLIC_KEYSTATIC_STORAGE=github npm run dev   # then open /keystatic/setup
```

## Adding & editing posts

Two ways, both commit to `src/content/posts/*.md`:

1. **Hosted CMS (any device):** https://rascunhos.blog/keystatic — log in with GitHub, add/edit, save. Saves commit to the repo and Vercel redeploys.
2. **Locally:** edit/create `src/content/posts/<slug>.md`, or run `npm run dev` and use `/keystatic` (local mode, no auth).

**Authoring caveats:**
- **Plain markdown only.** Astro renders `.md` as CommonMark; Markdoc tags (`{% %}`, `{#id}`) would render as literal text with no build error.
- **Don't change a published post's slug** (the filename) — the post URL `/posts/<slug>.html` would change with no redirect, breaking inbound links/RSS.
- A markdown thematic break (`---`) renders as the `⁂` asterism section break (via a rehype plugin in `astro.config.mjs`).

### Post frontmatter
```yaml
---
title: "the title"        # rendered as the <h1> (kept lowercase, by convention)
pubDate: 2026-02-27       # drives ordering + the RSS feed
excerpt: "…"              # optional; shown in listings/RSS
draft: false              # true = committed but hidden from the live site/feed
---
```

## Architecture / key files

| Path | Responsibility |
| --- | --- |
| `src/content/config.ts` | posts collection schema (Zod) |
| `src/content/posts/*.md` | the posts (source of truth) |
| `src/layouts/Base.astro` | shared chrome (head, footer, GoatCounter, favicon) |
| `src/pages/posts/[slug].html.astro` | post page — note the literal `.html` route preserves `/posts/<slug>.html` |
| `src/pages/index.astro`, `src/pages/page/[page].html.astro` | homepage (featured + recent + archive) + pagination |
| `src/pages/o-culpado/index.astro` | author page (`/autor.html` 301-redirects here, via `astro.config.mjs`) |
| `src/pages/feed.xml.ts` | RSS feed at `/feed.xml` |
| `keystatic.config.ts` | Keystatic CMS config (local in dev, GitHub mode in prod) |
| `astro.config.mjs` | static output + Vercel adapter + react + keystatic + the rehypeAsterism plugin + `/autor.html` redirect |

## Deployment (Vercel)

`main` auto-deploys to production (rascunhos.blog) via the Vercel project `rascunhos-blog`. `vercel.json` sets `framework: astro`.

**Keystatic env vars** (set in the Vercel project, Production + Preview; generated by the setup wizard, stored locally in the gitignored `.env`):
`KEYSTATIC_GITHUB_CLIENT_ID`, `KEYSTATIC_GITHUB_CLIENT_SECRET`, `KEYSTATIC_SECRET`, `PUBLIC_KEYSTATIC_GITHUB_APP_SLUG`. The GitHub App (`login-blog`) must stay installed on `hhpmacedo/rascunhos_blog`.

## Analytics & subscriptions
